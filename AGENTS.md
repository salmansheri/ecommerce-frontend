# AGENTS.md

Guidance for coding agents working in this repository.

## Project Overview

- Stack: React 19 + TypeScript + Vite + TanStack Start/Router.
- Styling: Tailwind CSS v4 and shadcn/ui primitives.
- Tooling: Biome for formatting/linting, Vitest for tests.
- Package manager: `pnpm` (lockfile is `pnpm-lock.yaml`).
- Path alias: `@/*` maps to `src/*`.

## Working Rules For Agents

- Prefer small, focused changes that match nearby conventions.
- Do not edit generated files unless the task is specifically about generation.
- Keep imports organized; run Biome checks before finalizing.
- Respect strict TypeScript settings (`strict`, no unused locals/params).

## Install / Dev Commands

- Install dependencies: `pnpm install`
- Start dev server: `pnpm dev`
- Build production assets: `pnpm build`
- Preview production build: `pnpm preview`

## Lint / Format / Check Commands

- Lint all configured files: `pnpm lint`
- Format all configured files: `pnpm format`
- Run Biome check (lint + format diagnostics): `pnpm check`
- Check a specific path with Biome: `pnpm check -- src/path/to/file.tsx`
- Lint a specific path with Biome: `pnpm lint -- src/path/to/file.tsx`
- Format a specific path with Biome: `pnpm format -- src/path/to/file.tsx`

## Test Commands (Vitest)

- Run all tests once: `pnpm test`
- Vitest direct entrypoint: `pnpm exec vitest`
- Run a single test file:
  - `pnpm exec vitest run src/foo/bar.test.ts`
  - or `pnpm test -- src/foo/bar.test.ts`
- Run tests matching a name/pattern:
  - `pnpm exec vitest run -t "renders product card"`
- Run in watch mode:
  - `pnpm exec vitest watch`
- If no tests exist yet, avoid CI failures with:
  - `pnpm exec vitest run --passWithNoTests`

## Build + Quality Gate Suggestion

Use this order before finishing substantial work:

1. `pnpm check`
2. `pnpm test` (or targeted test command)
3. `pnpm build`

## Source Layout Conventions

- `src/routes`: file-based TanStack routes.
- `src/components`: app and UI components.
- `src/components/ui`: shadcn-style reusable primitives.
- `src/lib`: utilities, static data, and helper modules.
- `src/integrations`: framework/tool integrations.
- `src/routeTree.gen.ts`: generated route tree (do not hand-edit).

## Generated Files

- `src/routeTree.gen.ts` is auto-generated by TanStack Router.
- It explicitly states it will be overwritten.
- Do not manually modify it.
- If route tree is stale, regenerate via normal TanStack tooling/workflow.

## TypeScript Guidelines

- Keep code fully type-safe; avoid `any` unless absolutely unavoidable.
- Prefer explicit interfaces/types for component props.
- Use type-only imports when importing types (`import type { X } ...`).
- Let inference work for obvious local values; annotate exported APIs.
- Keep strict-null safety in mind for optional/nullable values.
- Avoid unused variables and parameters (TS config enforces this).

## Import Guidelines

- Group imports in this order when practical:
  1. External packages
  2. Internal alias imports (`@/...`)
  3. Relative imports (`../...`, `./...`)
- Prefer alias imports for cross-feature/shared modules.
- Keep relative imports for same-folder or very local modules.
- Avoid deep, fragile relative chains when `@/` alias is clearer.
- Trust Biome organize-imports to normalize ordering where possible.

## Formatting Guidelines

- Biome is the source of truth for formatting.
- Indentation: tabs (configured in `biome.json`).
- JavaScript/TypeScript quote style: double quotes.
- Run `pnpm format` or `pnpm check` after edits.
- Do not hand-format against Biome output.

## Naming Conventions

- React components: PascalCase (`ProductCard`, `LoaderWrapper`).
- Hooks: `useXxx` naming.
- Route files: follow TanStack file-route naming conventions.
- Utility functions: descriptive camelCase verbs/nouns.
- Types/interfaces:
  - Existing code uses both `TType` and `IInterface` prefixes.
  - Preserve local file/module style; do not mass-rename unrelated code.

## React + UI Conventions

- Use functional components and typed props.
- Keep UI primitives in `src/components/ui` reusable and generic.
- Keep feature-specific logic in feature components/routes.
- Reuse existing utility helpers such as `cn` from `@/lib/utils`.
- Use shadcn patterns for new UI primitives.

## Data / Routing Conventions

- Define routes with TanStack route helpers (`createFileRoute`, etc.).
- Use route search validation with Zod where appropriate.
- Keep loader/action logic near corresponding route modules.
- Use typed route params/search data rather than ad-hoc parsing.

## Error Handling Guidelines

- Fail fast on invalid inputs or impossible states.
- Surface user-facing failures through route error boundaries/components.
- Reuse existing retry/error patterns (see router default error component).
- Avoid swallowing errors silently.
- Include contextual error messages when throwing new errors.

## Testing Guidelines

- Prefer colocated `*.test.ts` / `*.test.tsx` near tested modules.
- Use Testing Library for React component behavior.
- Test observable behavior, not implementation details.
- Keep test names descriptive and intent-focused.
- For bug fixes, add/regress a test when practical.

## Agent Checklist Before Handoff

- Run relevant Biome checks: `pnpm check` (or scoped path check).
- Run relevant tests (targeted or full).
- Ensure build is healthy for production-impacting changes.
- Confirm no manual edits were made to generated route tree files.
- Keep diffs minimal and aligned with nearby code patterns.

## Runtime & Hooks Gotchas

- TanStack Store hooks (`useStore`, `Derived`) must run in client components only. Using them in SSR shell/root components can cause `useRef is null` or `resolveDispatcher() is null` errors. If hooks fail in root layout/shell, use React's `useSyncExternalStore` instead.
- Dev server port `42069` (TanStack devtools event bus) may be in use. Kill existing Vite process if `EADDRINUSE` error occurs.

## Global Store Conventions

- Use `@tanstack/store` (`Store`, `Derived`) for app-wide state (see `src/lib/cart-store.ts`, `src/lib/auth-store.ts`).
- Pattern: create store, export it, optionally export a `Derived` for computed values, export setter helpers.
- Always use hooks in leaf components; avoid calling store methods outside React component flow.

## Header & Shell

- Header component (`src/components/header.tsx`) is rendered in `src/routes/__root.tsx` shell. It lives outside route transitions.
